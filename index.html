compiler vs interpreter

compiler란?

  1. 전체 소스코드를 읽어들여 명령어를 수집하고 그것을 재구성한다.
  2. 따로 기계어로 변환하는 빌드 과정이 필요하다.
  3. 컴파일 후에 실행파일을 생성한다.
  4. 소스코드를 변환하는 즉시 실행단계에 들어간다.
  5. 컴파일된 프로그램의 경우 runtime시 실행이 인터프리터보다 빠르게 동작한다.
  6. 실행파일을 생성해야 하기 때문에 수정 시 빌드 과정을 다시 거쳐야 한다.

compiler 언어 : C, Java, C#, C++

interpreter란?

  1. 소스코드의 각 행을 연속적으로 분석(읽어들여 변환)하여 실행한다.
  2. 기계어로 따로 변환하지 않고 해석한 즉시 바로 명령어를 실행한다. (빌드 시간이 존재하지 않는다.)
  3. 실행파일을 생성하지 않는다. (별도의 컴파일 시간이 없다.)
  4. 생성된 실행파일을 실행한다.
  5. 실행파일이 생성된 컴파일러보다 실행 속도가 느리다.
  6. 실행파일을 생성하지 않기 때문에 빌드 과정이 없어 수정 시 바로 실행이 가능하다.

interpreter 언어 : Javascript, Python, HTML(언어라고 본다면 포함된다.), R, SQL, ruby

191016 오늘의 질문

get과 set을 사용하는 이유?

객체의 데이터를 보호하기 위해서 일반적으로 다른 언어에서는 private라는 접근 제한자를 사용한다.
하지만 지금까지의 javascript에서는 이 접근 제한자가 없었고 접근 제한자를 구현하기 위해서 프로퍼티에 _를 붙여 암묵적으로 접근을 불허했다.

private를 사용하게 되면 원래는 외부에서 내부의 프로퍼티를 참조할 수 없게 되는데 이를 사용하기 위해서 get과 set을 사용한다.

get을 사용하게 되면 원본은 유지되고 return된 복사값만 수정이 되기 때문에 원본을 유지할 수 있다는 장점이 있다.

이걸 자세히 설명하기 위해서는 객체지향 프로그래밍의 원리에 대해서 알아야 한다. (추후 포스팅)
