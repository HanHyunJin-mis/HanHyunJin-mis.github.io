191016 오늘의 질문

get과 set을 사용하는 이유?

객체의 데이터를 보호하기 위해서 일반적으로 다른 언어에서는 private라는 접근 제한자를 사용한다.
하지만 지금까지의 javascript에서는 이 접근 제한자가 없었고 접근 제한자를 구현하기 위해서 프로퍼티에 _를 붙여 암묵적으로 접근을 불허했다.

private를 사용하게 되면 원래는 외부에서 내부의 프로퍼티를 참조할 수 없게 되는데 이를 사용하기 위해서 get과 set을 사용한다.

get을 사용하게 되면 원본은 유지되고 return된 복사값만 수정이 되기 때문에 원본을 유지할 수 있다는 장점이 있다.

이걸 자세히 설명하기 위해서는 객체지향 프로그래밍의 원리에 대해서 알아야 한다. (추후 포스팅)

Description

Alice and Bob have candy bars of different sizes: A[i] is the size of the i-th bar of candy that Alice has, and B[j] is the size of the j-th bar of candy that Bob has.

Since they are friends, they would like to exchange one candy bar each so that after the exchange, they both have the same total amount of candy. (The total amount of candy a person has is the sum of the sizes of candy bars they have.)

Return an integer array ans where ans[0] is the size of the candy bar that Alice must exchange, and ans[1] is the size of the candy bar that Bob must exchange.

If there are multiple answers, you may return any one of them. It is guaranteed an answer exists.

[191001 LeetCode] Fair Candy Swap

Description

We are given a matrix with R rows and C columns has cells with integer coordinates (r, c), where 0 <= r < R and 0 <= c < C.

Additionally, we are given a cell in that matrix with coordinates (r0, c0).

Return the coordinates of all cells in the matrix, sorted by their distance from (r0, c0) from smallest distance to largest distance. Here, the distance between two cells (r1, c1) and (r2, c2) is the Manhattan distance, |r1 - r2| + |c1 - c2|. (You may return the answer in any order that satisfies this condition.)
