절차지향, 객체지향

절차지향, 객체지향 프로그래밍

절차지향이란?

절차지향에서는 물이 위에서 아래로 흐르듯이 순차적인 처리가 기본이 된다.

이는 프로그램 전체가 유기적으로 연결되도록 만드는 프로그래밍 기법 또는 패러다임이라고 할 수 있다.

절차지향에서는 코드의 실행 순서가 위에서 아래로 정해져 있으므로 코드 순서가 바뀌게 되면 실행 결과가 달라진다.
그리고 코드가 컴퓨터의 작업 처리 방식과 유사하기 때문에 객체지향보다는 더 빨리 처리된다고 볼 수 있다.

데이터(즉, 구조체)와 함수(기능)가 분리되어 있다.

프로그램의 순서와 흐름을 먼저 세우고 필요한 자료구조와 함수들을 설계하는 방식

객체지향이란?

개발하려는 것을 지능별로 묶어 모듈화를 하는 것

-> 하드웨어가 같은 기능을 중복으로 연산하는 것이 아니라 모듈을 재활용함으로써 하드웨어의 처리양을 획기적으로 줄여줄 수 있다.

객체지향의 4가지 특징

  1. 추상화

    공통의 속성이나 기능을 묶어 이름을 붙이는 것
    ex. 인터페이스, 구조체, class

  2. 캡슐화

    하나의 기능을 위해 묶인 class 내에서만 통용되는 변수와 메소드를 외부에서 접근할 수 없도록 차단하는 것
    즉, 데이터를 감추어 외부 상호작용은 메소드를 통해서 하는 방법 (get / set)

  3. 상속성

    상위 개념의 특징을 하위 개념이 물려받는 것
    공통된 소스코드를 부모 class로 정의하고 자식은 이를 상속받아 소스의 재사용율을 높인다.
    ex. prototype, class

  4. 다형성

    overriding, 가공하여 여러 타입으로 변형 가능함을 의미한다.
    super, extends

모듈에 있는 기능 중 하나만 필요하더라도 전체를 가져온다는 면에서 크기가 커질 수 있다.

객체 지향 프로그래밍을 하기 위해서는 객체의 설계가 우선되며 자료구조, 모듈들을 우선 설계한 후 실행 순서나 흐름을 짜야 한다.

하지만 그렇다고 해서 객체 지향 프로그래밍이 절차를 무시하는 것은 아니다.
또한 둘은 상반된 개념이 아니라 단지 다른 개념이라고 이해하는 것이 좋다.
